# 12 — Wings Node Setup Guide

> **"Publish node too and Connect it too"** — this guide walks through
> creating a Pterodactyl node (Wings daemon), connecting it to your
> decoupled panel, and verifying the full stack.

## 1. Architecture recap

```
┌────────────────────────┐         ┌─────────────────────────────┐
│  Vercel (frontend)     │         │  Linux server (backend)     │
│  deathlegionpanel      │  HTTPS  │  api.yourdomain.com         │
│  .vercel.app           │ ──────► │  Laravel + MySQL + Redis    │
└────────────────────────┘         └─────────────┬───────────────┘
                                                  │ Panel → Wings
                                                  │ Bearer {decrypted
                                                  │   daemon_token}
                                                  ▼
                                    ┌─────────────────────────────┐
                                    │  Node server (Wings)        │
                                    │  node1.yourdomain.com:8080  │
                                    │  runs game containers       │
                                    └─────────────────────────────┘
```

The Wings daemon is **unchanged** from upstream. You install it the same
way you would for any Pterodactyl panel — the panel just happens to be
the decoupled version.

## 2. Prerequisites for the node

- A Linux server (Ubuntu 22.04+ recommended) with:
  - Docker installed (`curl -fsSL https://get.docker.com | sh`)
  - At least 2GB RAM + 20GB disk
  - Ports 8080 (Wings HTTP) and 2022 (SFTP) open in the firewall
  - A public FQDN resolvable to this server (e.g. `node1.yourdomain.com`)

## 3. Create the node in the panel

### 3.1 Open the admin area

The admin area is served by the backend at `https://api.yourdomain.com/admin`.
Log in with your admin account.

### 3.2 Add a location

`Admin → Locations → Create Location` (e.g. "US East").

### 3.3 Add the node

`Admin → Nodes → Create Node`:
- **Name**: `node1`
- **Location**: select the location you just created
- **FQDN**: `node1.yourdomain.com` (must resolve via DNS)
- **Communicate over SSL**: ✅ (you'll get a TLS cert via Let's Encrypt below)
- **Behind Proxy**: ✅ if you put Wings behind Cloudflare/etc., else ❌
- **Memory / Memory Overallocate / Disk / Disk Overallocate**: your capacity
- **Daemon Port**: `8080`
- **Daemon SFTP Port**: `2022`
- **Daemon Base Directory**: `/var/lib/pterodactyl/volumes` (default)

Save. The panel auto-generates a `daemon_token_id` and an encrypted
`daemon_token` — these are stored in the `nodes` table and shown in the
node's **Configuration** tab.

### 3.4 Allocate ports

`Admin → Nodes → node1 → Allocations`:
- Add ports like `25565-25575` (Minecraft), `27015-27025` (SRCDS), etc.
- These are the ports servers can bind to.

## 4. Install Wings on the node

### 4.1 Get the Wings binary

```bash
# On the node server:
sudo mkdir -p /etc/pterodactyl /var/lib/pterodactyl/volumes
sudo chown -R root:root /etc/pterodactyl

# Download the latest Wings binary
sudo curl -L -o /usr/local/bin/wings https://github.com/pterodactyl/wings/releases/latest/download/wings_linux_amd64
sudo chmod u+x /usr/local/bin/wings
```

### 4.2 Download the node config

From the panel admin:
`Admin → Nodes → node1 → Configuration → Download configuration`

Upload that `config.yml` to `/etc/pterodactyl/config.yml` on the node server.

The file looks like:
```yaml
debug: false
api:
  host: 0.0.0.0
  port: 8080
  ssl:
    enabled: true
    cert: /etc/letsencrypt/live/node1.yourdomain.com/fullchain.pem
    key: /etc/letsencrypt/live/node1.yourdomain.com/privkey.pem
  upload_limit: 100
system:
  data: /var/lib/pterodactyl/volumes
  sftp:
    bind_port: 2022
  user:
    root: true
remote: https://api.yourdomain.com
token_id: <auto-generated>
token:   <decrypted daemon_token — panel handles this>
```

**IMPORTANT**: The `token_id` and `token` in `config.yml` are generated by
the panel and shown in the admin UI. Do NOT edit them manually. If you
regenerate the token in the panel, re-download the config.

### 4.3 TLS for Wings

Wings needs TLS to talk to the panel (which is HTTPS). Easiest: issue a
Let's Encrypt cert for the node's FQDN:

```bash
sudo certbot certonly --standalone -d node1.yourdomain.com --non-interactive --agree-tos -m admin@yourdomain.com
```

(The standalone plugin needs port 80 free temporarily.)

Then in `config.yml`, point `ssl.cert` and `ssl.key` to the Let's Encrypt
paths shown above.

### 4.4 Run Wings as a systemd service

```bash
sudo curl -L -o /etc/systemd/system/wings.service https://raw.githubusercontent.com/pterodactyl/wings/develop/install/wings.service
sudo systemctl daemon-reload
sudo systemctl enable --now wings
sudo systemctl status wings  # verify it's running
sudo journalctl -u wings -f  # live logs
```

You should see `Wings is now listening on 0.0.0.0:8080`.

### 4.5 Verify panel → node connectivity

In the panel admin:
`Admin → Nodes → node1 → Health`

You should see:
- Node health: ✅
- Memory / Disk / CPU usage reported
- Wings version reported

If this fails, check:
- Wings is running on the node (`sudo systemctl status wings`)
- The node's FQDN resolves from the panel server
- Port 8080 is reachable from the panel server (`nc -zv node1.yourdomain.com 8080`)
- TLS cert is valid (not expired, hostname matches)

## 5. Verify end-to-end with the frontend

### 5.1 Update the frontend's API URL

In Vercel dashboard:
`deathlegionpanel → Settings → Environment Variables → VITE_API_URL`
→ set to `https://api.yourdomain.com`

Then trigger a redeploy:
`deathlegionpanel → Deployments → ⋮ → Redeploy`

### 5.2 Set CORS + Sanctum on the backend

In `/var/www/pterodactyl/.env`:
```
SANCTUM_STATEFUL_DOMAINS=deathlegionpanel.vercel.app,panel.yourdomain.com
CORS_ALLOWED_ORIGINS=https://deathlegionpanel.vercel.app,https://panel.yourdomain.com
SESSION_DOMAIN=.vercel.app   # or .yourdomain.com if frontend is on your domain
SESSION_SAMESITE=none
SESSION_SECURE_COOKIE=true
```

Then:
```bash
cd /var/www/pterodactyl
php artisan config:cache
sudo supervisorctl restart pterodactyl-worker:*
```

### 5.3 Test the full flow

1. Visit `https://deathlegionpanel.vercel.app` → login screen.
2. Log in with your admin user.
3. `Dashboard` → should see your servers list.
4. `Admin → Servers → Create Server` (in a new tab — `https://api.yourdomain.com/admin`):
   - Select node1, an egg (Minecraft etc.), allocate resources.
   - Click create.
5. Back in the frontend dashboard → server card should appear.
6. Click the server → Console tab → wait for `Connected`.
7. Click `Start` → watch logs stream in.
8. Type a command in the input (e.g. `list` for Minecraft) → press Enter.

If all of that works, your decoupled panel is fully operational. 🎉

## 6. Common issues

| Symptom | Cause | Fix |
|---------|-------|-----|
| `Network Error` on login | CORS or API URL wrong | Verify `VITE_API_URL` in Vercel env + `CORS_ALLOWED_ORIGINS` includes the Vercel URL |
| `419 CSRF token mismatch` | Cookie not flowing cross-origin | Set `SESSION_DOMAIN` + `SESSION_SAMESITE=none` + `SESSION_SECURE_COOKIE=true` |
| Console won't connect | Wings unreachable from browser | Verify node FQDN + port 8080 + TLS cert from your browser's network |
| Console connects but no output | JWT expired | Wait — the frontend auto-refreshes on `token expiring`. If it persists, check `NodeJWTService` config. |
| `daemon connection refused` in panel admin | Wings can't reach panel | Check `remote:` in node `config.yml` is `https://api.yourdomain.com` |
| Server stuck in `installing` | Wings install script failing | Check `sudo journalctl -u wings -f` on the node |
| File upload 401 | Direct-to-Wings JWT expired | Re-fetch the upload URL (15-min TTL) |

## 7. Multiple nodes

To add more nodes, repeat steps 3-4 for each. Each node gets its own:
- FQDN (e.g. `node2.yourdomain.com`)
- TLS cert
- Wings daemon_token (panel auto-generates)
- Allocations

The panel handles routing servers to nodes automatically based on the
node assignment set when the server is created.

## 8. Updating Wings

When a new Wings version is released:
```bash
sudo systemctl stop wings
sudo curl -L -o /usr/local/bin/wings https://github.com/pterodactyl/wings/releases/latest/download/wings_linux_amd64
sudo chmod u+x /usr/local/bin/wings
sudo systemctl start wings
```

No panel-side changes needed — the Panel → Wings HTTP surface is stable.
